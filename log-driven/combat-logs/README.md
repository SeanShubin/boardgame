# Combat logs

Worked, physical-card combat walkthroughs for deckbound — plus the shared legend and the
tutorial curriculum. Three buckets by purpose:

- **`combat-reference.md`** — the **legend**: every stat, the six phases, the cast/resolve
  timing model, and every board symbol. **Read this first.**

- **`designer/`** — dense **designer** logs (verification & validation): many mechanics per
  battle, built so the *designer* can spot lost nuance. Not tutorials.
  - `card-combat-all-mechanics.md` — the grand tour (every mechanic, action by action); also
    the tutorial **capstone**.
  - `card-combat-round-4v4.md` — a front-holds attrition round (groups, spillover, responses).
  - `card-combat-round-breach.md` — a breakthrough round (per-unit lock, pre-empt, disrupt).
  - `mechanic-validation-six-phase.md` — isolated one-mechanic checks (pre-empt / groups / flank).

- **`tutorials/`** — the ordered **player-tutorial** series: **one new mechanic per log**, each
  relying only on earlier lessons.
  - `00-sequence.md` — the curriculum in dependency order; **start here**.
  - `01-…` onward — the lessons (written in order).

Related: the tutorial **philosophy** is `../brainstorming/tutorial-design.md`; the
necessity-test tie-in is `../../docs/games/deckbound/computability-and-balance.md` §6.1.
